/* What is local storage? */

  • Type of web storage storing data as key-value pairs that can be accessed via JavaScript
  • Website and browser specific data that is stored locally on userâ€™s machine with no expiration date.
  • The data will persist even after the browser window has been closed.

/* Working with local storage */

As can be seen from the code sample, the interface for accessing and modifying contents of local storage is fairly simple. To learn more about local storage and its sibling technology session storage, visit MDN web docs on Web Storage API.

// Set a key-value pair
localStorage.setItem('users-name', nameValue)

// Access a value using key
localStorage.getItem('users-name')

// Delete a key-value pair
localStorage.removeItem('users-name')

// Remove all key-value pairs (associated with this domain)
localStorage.clear()

Key-value pairs are always stored as strings (source w3schools), but that does not mean that one is limited to only string data. When using other formats, such as numbers, one just needs to remember to convert the stored string value to right data type.

Local storage can also be used to store JavaScript objects by simply conversting the object to a JSON string. Then to use or update the object, the JSON string can be conververted back to a JavaScript object.

// Define an example JavaScript object
let user = {
firstName: "Harry",
lastName: "Potter",
email: "harry.potter@hogwarts.edu",
alsoKnownAs: ["Boy Who Lived", "Chosen One", "Undesirable No. 1"],
wand: {
length: 11,
wood: "holly",
code: "phoenix feather"
}
}

// Convert an object to string for storage and store it
let userString = JSON.stringify(user);
localStorage.setItem('user-details', userString)

// Convert retrieved string back to a JavaScript object
let parsedObject = JSON.parse(localStorage.getItem ('user-details'))

/* How to examine local storage data in browser? */

The data that a specific website is storing to local storage ban be viewed using browser's developer tools.

Chrome

Starting from the top bar menu, follow the path below to view local storage data (associated with currently visited domain)

  1. View >> Developer >> Developer Tools
  2. Select tab 'Application' (you may need to click arrow >> to reveal this and other options)
  3. From side panel section 'Storage' open 'Local Storage'
  4. Below local storage is one or more domains for which the local storage data has been stored.
Firefox

Starting from the top bar menu, follow the path below to view local storage data (associated with currently visited domain)

Tools >> Web Developer >> Storage Inspector

Then select local storage to view local storage data.

Edge

Starting from the top bar menu, follow the path below to view local storage data (associated with currently visited domain)

  1. Tools >> Developer >> Developer Tools...
  2. Select tab 'Application' (you may need to click arrow >> to reveal this and other options)
  3. From side panel section 'Storage' open 'Local Storage'
  4. Below local storage is one or more domains for which the local storage data has been stored.
Opera

Starting from the top bar menu, follow the path below to view local storage data (associated with currently visited domain)

  1. From top menu select Developer >> Developer Tools
  2. Select tab 'Application' (you may need to click arrow >> to reveal this and other options)
  3. From side panel section 'Storage' open 'Local Storage'
  4. Below local storage is one or more domains for which the local storage data has been stored.
Safari

If the top menu bar already has an option 'Developer', skip to step 4.

  1. Safari >> Preferences >> Advanced
  2. Check the checkbox "Show Developer menu in menu bar"
  3. Now top menu bar has option 'Develop'
  4. Develop >> Show Web Inspector
  5. Select tab 'Storage' and then 'Local storage

/* Local storage data on your machine */

Since local storage data is stored locally on your machine, even if your browser does not provide sufficient tools and access for viewing and deleting local storage data, you can dig into that data in your file browser.

On Windows

Chrome

For Google Chrome users on Windows, you can access the local storage data from the following file path:

%LocalAppData%\Google\Chrome\User Data\Default\Local Storage

In the subfolder, you should find a LOG file that will contain information on the user's actions, such as, generating tables and deleting data types. The time of the activity is also logged in chronology.

/* Local storage and private mode */

Using private mode (also known as incognito mode), essentially turn local storage into session storage. Local storage data can be created and updated, it persist through page refreshes and closing and re-opening tabs, but is deleted once you close the private window. (All the browsers listed in the table above behaved this way in their own private mode.)
